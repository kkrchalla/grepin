Configure your site for indexing
In this page you can enter the details of your web site that will help our robot to index your web site.
You can access this page by logging in to your account and clicking on the "Configure Your Site Indexing" link in the "Member's Home" page or on the "site indexing" link in the left navigation bar.
Enter the appropriate data in the fields of this page and click the "Submit Form" button.
Then your site will be queued for indexing.

Descriptions of all the fields are given below:

Base URL
This is the domain name of your Web Site like http://www.yoursite.com/.
This URL should start with 'http://' or 'https://' and end with a '/'.

Index start URL
This is the URL of the page in your Web Site where the indexer should start crawling. This could be your home page or your site map page.
This URL should start with 'http://' or 'https://' and end with a '/'.
For example:
http://www.yoursite.com/index.html/ or
http://www.yoursite.com/somepage.html/
Maximum number of pages
A rough estimate of the total number of pages in your Web Site. Pick one entry from the dropdown list box.

Limit indexing to following URLs
Only the pages starting with these URLs will be indexed. If you only want the pages starting with your Base URL to be indexed, do not enter anything in this field. But if you want the pages that do not start with your Base URL to be indexed, you have to enter those URLs, like the subdomains (ex: http://subdomain.yoursite.com/). Enter as many URLs as needed separated by at least one space.
These URLs should start with 'http://' or 'https://' and end with a '/'.
For example:
http://songs.yoursite.com/
http://www.somesite.net/
Note: The data in this field cannot exceed 255 characters in length.

Exclude the following pages or paths from indexing
If you do not want your visitors to find certain pages during searching, you can exclude them from being indexed. You can either give the full url of a page (like http://www.yoursite.com/idx.html/) or, you can give the partial url (like /somepath/) and all the pages that contain this partial text will be excluded.
    Following are some examples of how to exclude your pages from being indexed.
  1. http://www.yoursite.com/restricted.html/
    This will exclude page
    "http://www.yoursite.com/restricted.html/".
  2. /restricted/
    This will exclude pages like
    http://www.yoursite.com/restricted/somepage.html
    http://www.yoursite.com/somepath/restricted/somepage.html
  3. /somepath/somepages*
    This will exclude pages like
    http://www.yoursite.com/somepath/somepages.html
    http://www.yoursite.com/somepath/somepages.xml
    http://www.yoursite.com/path1/path2/somepath/somepagesmore.html
  4. /somepath/*nav.html
    This will exclude pages like
    http://www.yoursite.com/somepath/nav.html
    http://www.yoursite.com/somepath/billnav.html
    http://www.yoursite.com/path1/path2/somepath/somenav.html
You can give as many as you want separating them by at least one space like
http://www.yoursite.com/somepage.html /path/ /path1/page* /path2/*nav.html

Note: The data in this field cannot exceed 255 characters in length.

Common terms to be excluded from indexing
These are the terms that appear a lot on your web site and has no meaning for your visitor to search on.
These are the terms like your web site name, or navigational link names like 'Home', etc., and also your web site captions that appear on almost all of your web pages.
Enter as many terms or words as you wish, each separated by at least one space.

Note: The data in this field cannot exceed 255 characters in length.

Advanced
This method is for those who are comfortable with HTML. This will give you a great flexibility to build your search results page to look and feel like one of your own web pages.
In this method, you can directly type the HTML code for the search results page in the text box of this page.
You have to give the string :::search-results::: in your HTML code where you want the search results to appear.
The Grepin Search program will then display the search results in your page where this string occurs.

Once you type in the HTML, click the "Preview Results Page" button to see how your search results page will look.
If you are satisfied with the look of this page, click "Confirm the Settings" button on the preview page to confirm your search results page settings.

When you code the HTML, be sure to use absolute path (fully qualified) for all the URLs for links and images. Relative URLs will not work here as they will refer to Grepin.com.
An example of absolute URL is "http://www.yoursite.com/pagelink.html".

An example:

<html>
<body>
.
.
HTML code of your page ...
.
.
:::search-results:::
.
.
HTML code of your page ...
.
.
</body>
</html>

View search activity reports
You can the search activity on your web site using the 3 different types of reports.

Monthly report will give you a consolidated look at the search activity on your web site in a month.

Detailed

This is the detailed report of the search activity on your web site in the last 7 days.
Description of the fields:
Search Terms - The terms searched by your visitors.
Search Time - The date and time when this search was performed.
Duration - The time (in seconds) it took to perform this search.
Num of Results - Total number of results brought by this search.
Next Prev - Yes = This search was performed by clicking the Next, Prev, or the page number links on your search results page.
Source - Gives you the name of the search box on your Web Site that originated this search. In order to use this feature you should add 'source' option to your search box HTML.
Monthly
This is the consolidated report of the search activity on your web site for a month.
Click on the desired month to view the report of that particular month.
Description of the fields:
Search Terms - The terms searched by your visitors.
Num of Times Searched - Total number of times (including Next, Prev) the 'Search Terms' were searched by your visitors in the specific month.
Next Prev - Total number of times the 'Search Terms' were searched by your visitors in the specific month, using the 'Next', 'Prev', or specific page number links on your search results page.

Description of the fields:

How to build pages that are perfect for searching and displaying in search results?
  1. A good title in title tag and a meaningful description in meta-description tag will help your visitor identify a page in the search results.
  2. Putting the main keywords in title and description will boost the ranking of the page.
  3. Putting the main keywords in the Main-Heading and Sub-Headings of the page will boost the ranking of the page.
How can I eliminate a portion of a page from being indexed?
You can use ignore tags to eliminate a portion of a page from being indexed.
In the HTML of the page put the <!--ignore-grepin-start--> at the beginning of the section you would like to eliminate and <!--ignore-grepin-end--> at the end of it.

For example:

<!--ignore-grepin-start-->
.
.
you html code to be excluded from indexing
.
.
<!--ignore-grepin-end-->

How can I prevent a page from being indexed?
Code the robot meta tag as follows:
<META NAME="robots" CONTENT="noindex">
How can I prevent Grepin Site Search from following the links in a page?
Code the robot meta tag as follows:
<META NAME="robots" CONTENT="nofollow">
Where should I place the search box in my web site?
  1. Definitely in the home page.
  2. And in every page. This could also be a hyperlink that takes them to a separate page that contains the search box.
  3. You do not need to always use a search box. You can give a direct link on your web page which when user clicks will show the results of a particular term search. For example, you can give a link like "Click to see all the kinds of boots we carry". This link could be the link to search for the term "boots" on your website.
How can I promote my other web sites in a subtle manner?
  1. Have at least one link leading from one web site to another.
  2. Then use the "Limit indexing to following URLs" field in the "Configure Your Site Indexing" page to give the Base URLs of all of your web sites.
  3. Then use include, or exclude option in the search box HTML to get the results you want.

How does the Grepin Site Search work?
    Grepin Site Search is made of 2 components.

  1. Grepinbot - a robot that crawls through your web site indexing all the content in your web pages. It reads the content of your page, parses it, indexes it in our index tables on our servers.

  2. Grepin Search - a search program that scans the index tables to retrieve the information requested by your visitor through a search box. It will first rank the results based on our algorithm. Then it will sort the results, and display them on the screen in a search results page configured by you. It also logs your visitor's search query in a log database.
